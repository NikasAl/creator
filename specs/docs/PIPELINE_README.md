# Pipeline Scripts

## Новая архитектура

Теперь все скрипты pipeline объединены в единую систему с конфигурационными файлами.

### Основные файлы:

- `run_pipeline.sh` - основной скрипт с общей механикой
- `configs/` - директория с конфигурационными файлами
- `run_stalin.sh`, `run_lem.sh`, `run_lomonosov.sh` - удобные скрипты-обертки

### Использование:

#### Вариант 1: Использование скриптов-оберток
```bash
./run_stalin.sh      # Запуск pipeline для проекта Сталин
./run_lem.sh         # Запуск pipeline для проекта Лем  
./run_lomonosov.sh   # Запуск pipeline для проекта Ломоносов
```

#### Вариант 2: Прямое использование основного скрипта
```bash
./run_pipeline.sh configs/stalin.conf
./run_pipeline.sh configs/lem.conf
./run_pipeline.sh configs/lomonosov.conf
```

### Конфигурационные файлы:

Каждый конфигурационный файл содержит параметры для конкретного проекта:

- `PDF_FILE` - путь к PDF файлу
- `OUTPUT_DIR` - выходная директория
- `PAGE_RANGE` - диапазон страниц
- `SUMMARY_STYLE` - стиль пересказа (educational, story, folk)
- `TITLE` - заголовок проекта
- `AUTHOR` - автор
- `PARTS` - количество частей/иллюстраций
- `STYLE` - стиль иллюстраций
- `PLATFORM` - платформа для публикации (YouTube, RuTube)

### Преимущества новой архитектуры:

1. **Устранение дублирования** - вся механика в одном файле
2. **Легкость поддержки** - изменения в одном месте
3. **Простота добавления новых проектов** - достаточно создать новый конфигурационный файл
4. **Исправлены проблемы с путями** - все Python скрипты используют абсолютные пути
5. **Добавлено создание HTML** - автоматическое создание HTML из промо-описания с паузой для редактирования

### Добавление нового проекта:

1. Создать конфигурационный файл в `configs/new_project.conf`
2. Создать скрипт-обертку `run_new_project.sh`
3. Сделать скрипт исполняемым: `chmod +x run_new_project.sh`
